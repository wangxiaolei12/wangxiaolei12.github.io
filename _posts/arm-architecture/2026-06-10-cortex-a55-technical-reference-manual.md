## 1. 关于 Cortex-A55 核心

Cortex-A55 核心是一款中端低功耗核心，实现了 Armv8-A 架构，支持以下扩展：

- Armv8.1-A 扩展
- Armv8.2-A 扩展
- RAS 扩展
- Armv8.3-A 的 Load acquire (LDAPR) 指令
- Armv8.4-A 的点积 (dot product) 指令

核心拥有一级 (L1) 内存系统和私有二级 (L2) 缓存。核心被实现在 **DynamIQ Shared Unit (DSU)** 中，作为 Little 核心，可与其他核心高度配置。

## 2. 特性

### 核心特性

- 完整实现 Armv8.2-A A64、A32 和 T32 指令集
- 在所有异常级别 (EL0 到 EL3) 支持 AArch32 和 AArch64 执行状态
- **有序 (in-order) 流水线**，带直接和间接分支预测
- 独立的 L1 数据和指令侧内存系统，带 MMU
- 支持 Arm TrustZone 技术
- 可选的数据引擎单元 (Data Engine)，实现 Advanced SIMD 和浮点架构
- 可选的加密扩展（仅在 Data Engine 存在时可用）
- GIC CPU 接口，连接外部分发器
- 通用定时器接口，支持来自外部系统计数器的 64 位计数输入

### 缓存特性

- 可选的统一私有 L2 缓存
- L1 和 L2 缓存保护（ECC 或奇偶校验）

### 调试特性

- RAS 扩展
- Armv8.2-A 调试逻辑
- 性能监视单元 (PMU)
- 嵌入式跟踪宏单元 (ETM)，仅支持指令跟踪

### 指令获取单元 (IFU)

IFU 从指令缓存或外部内存获取指令，并预测指令流中分支的结果。它将指令传递给数据处理单元 (DPU) 进行处理。

### 数据处理单元 (DPU)

DPU 解码和执行指令。它通过与数据缓存单元 (DCU) 接口来执行需要与内存系统进行数据传输的指令。DPU 包括 PMU、Advanced SIMD 和浮点支持，以及加密扩展。

**PMU**：提供 6 个性能监视器，可配置为收集每个核心和内存系统操作的统计信息。

**Advanced SIMD 和浮点支持**：Advanced SIMD 是一种媒体和信号处理架构，主要为音频、视频、3D 图形、图像和语音处理添加指令。浮点架构提供对单精度和双精度浮点运算的支持。

**加密扩展**（可选）：加速以下算法：
- AES 加密和解密
- SHA-1、SHA-224、SHA-256 哈希函数
- 有限域算术（用于 GCM 和 ECC 等算法）

## 4. 内存管理单元 (MMU)

MMU 通过一组虚拟到物理地址映射和保存在转换表中的内存属性来提供细粒度内存系统控制。这些条目在地址被转换时保存到 TLB 中。

TLB 条目包括：
- 全局和地址空间标识符 (ASID)——防止上下文切换时刷新 TLB
- 虚拟机标识符 (VMID)——防止 hypervisor 进行虚拟机切换时刷新 TLB

### TLB 组织

```
┌───────────────────────────────────────────────────────────────┐
│                         MMU                                     │
│                                                                 │
│  ┌──────────────┐    ┌──────────────┐                          │
│  │ L1 指令 TLB  │    │ L1 数据 TLB  │   ← 全关联              │
│  │  (10 条目)   │    │  (10 条目)   │                          │
│  └──────┬───────┘    └──────┬───────┘                          │
│         │    未命中          │    未命中                         │
│         └────────┬───────────┘                                  │
│                  ▼                                               │
│  ┌────────────────────────────────┐                             │
│  │        统一 L2 TLB             │   ← 4路组相联              │
│  │     (1024 条目, 奇偶校验)      │                             │
│  └────────────────┬───────────────┘                             │
│                   │ 未命中                                       │
│                   ▼                                              │
│  ┌────────────────────────────────┐                             │
│  │     IPA Cache RAM              │   ← Stage 2 IPA 缓存       │
│  └────────────────────────────────┘                             │
│  ┌────────────────────────────────┐                             │
│  │     Walk Cache RAM             │   ← 中间级别转换缓存        │
│  └────────────────────────────────┘                             │
│                   │ 未命中                                       │
│                   ▼                                              │
│           硬件页表遍历                                           │
└───────────────────────────────────────────────────────────────┘
```

**L1 TLB：**
- 指令和数据侧各自实现第一级 TLB 缓存
- 所有 TLB 相关维护操作会刷新指令和数据 L1 TLB

**L2 TLB：**
- 统一的 L2 TLB 处理 L1 TLB 的未命中
- 在带有核心缓存保护的实现中，奇偶校验位保护 TLB RAM，检测到错误时条目被无效化并重新获取

**IPA Cache RAM：** 缓存 Stage 2 的 IPA 转换中间结果。

**Walk Cache RAM：** 缓存多级页表遍历的中间级别结果，减少完整遍历次数。

### 转换表遍历

**AArch64 行为：**
- MMU 支持 4KB、16KB 和 64KB 转换粒度
- 支持最多 4 级转换表查找
- 支持硬件管理 Access flag 和 dirty state

## 5. L1 内存系统

### 概述

L1 内存系统包括：
- 数据缓存单元 (DCU)
- 存储缓冲区 (STB)
- 总线接口单元 (BIU)

**DCU** 管理所有 load 和 store 操作。L1 数据缓存 RAM 使用 ECC 保护（SECDED 方案：单错误纠正，双错误检测）。DCU 包含组合的本地和全局 exclusive 监视器。

**STB** 保存已从 load/store 流水线离开并由 DPU 提交的 store 操作。STB 可以请求访问 L1 数据缓存、发起行填充，或写入 L2/L3 内存系统。

**BIU** 包含到 L2 内存系统的接口和缓冲区。

### 缓存行为

**指令缓存禁用行为：** 当指令缓存被禁用时，所有指令获取都直接从外部内存进行。

**数据缓存禁用行为：** 当数据缓存被禁用时，所有数据访问都直接从外部内存进行。可缓存的 Normal 内存访问变为 Non-cacheable。

**数据缓存一致性：** L1 数据缓存参与 DSU 管理的一致性协议。

**Write Streaming Mode：** 当检测到连续写入模式时，DCU 可以绕过缓存分配，直接将数据写入下一级内存系统，避免不必要的行填充。

**复位时数据缓存无效化：** 冷复位后，数据缓存的状态未定义。在使能缓存之前，软件必须使用无效化操作。

### L1 指令内存系统

**程序流预测：** Cortex-A55 使用分支预测来减少流水线停顿。预测器包含：
- 分支目标缓冲区 (BTB)
- 全局历史缓冲区
- 返回栈

### L1 数据内存系统

**Internal Exclusive Monitor：**
- 用于 Load-Exclusive 和 Store-Exclusive 指令
- 如果在 Load-Exclusive 和 Store-Exclusive 之间有另一个 store 到同一地址，监视器会清除独占状态

## 6. L2 内存系统

L2 缓存是**可选的**，每个核心私有。特性：

- **4 路组相联**
- 支持 **64 字节缓存行**
- 可配置 RAM 大小：**64KB 到 256KB**
- 通过可选异步桥连接到 DSU

L2 内存系统支持与 L1 相同的内存类型（Normal 可缓存/不可缓存，Device）。

## 7. 电源管理

### 概述

Cortex-A55 提供动态和静态功耗控制机制：

**动态功耗管理：**
- 架构级时钟门控
- 每核 DVFS（动态电压频率调节）

**静态功耗管理：**
- 动态保持 (dynamic retention)
- 掉电 (powerdown)

### 电压域

```
┌─────────────────────────────────────────────────────────────┐
│                    每个 Cortex-A55 核心                       │
│                                                              │
│  ┌──────────────────────────────────────┐                   │
│  │          VCPU 电压域                  │ ← 核心处理逻辑    │
│  │  (核心逻辑 + 异步桥核心侧)           │    支持独立 DVFS   │
│  └──────────────────────────────────────┘                   │
│                                                              │
│  ┌──────────────────────────────────────┐                   │
│  │          VSYS 电压域                  │ ← DSU 侧         │
│  │  (异步桥 DSU 侧)                    │                   │
│  └──────────────────────────────────────┘                   │
└─────────────────────────────────────────────────────────────┘

注意：如果不需要 DVFS，可以将 VCPU 和 VSYS 绑定到同一电源。
```

### 电源域

| 电源域 | 描述 |
|--------|------|
| PDCPU\<n\> | 包含所有核心处理逻辑、异步桥的 CPU 时钟域逻辑、Advanced SIMD/FP 块、L1/L2 TLB、L1/L2 核心 RAM、调试寄存器 |
| PDADVSIMD\<n\> | 可选，Advanced SIMD/FP 块的独立电源域（实现 SIMD 动态保持） |
| PDSYS | 包含集群时钟域的异步桥逻辑 |

### 电源模式

| 模式 | 描述 |
|------|------|
| On | 核心正常运行，全时钟、全功能 |
| Off | 核心完全关电，所有状态丢失 |
| Emulated Off | 逻辑关电但保持调试寄存器 |
| SIMD dynamic retention | SIMD/FP 块时钟门控但保持状态 |
| Core dynamic retention | 核心时钟门控但保持状态 |
| Debug recovery | 核心关电后恢复调试连接 |

### WFI（等待中断）

当核心执行 `WFI` 指令时：
1. 核心等待所有指令（包括显式内存访问）退出
2. 确保 store 指令已更新缓存或已发送到 L3 内存系统
3. 进入低功耗状态（禁用核心中大部分时钟，核心仍然上电）

在 WFI 低功耗状态下，以下事件会临时使能时钟但**不会**退出 WFI：
- L3 snoop 请求
- 缓存或 TLB 维护操作
- APB 访问调试/跟踪寄存器
- GIC CPU 通过 AXI4 流通道访问

退出 WFI 的条件：
- 检测到 WFI 唤醒事件（如中断）
- 检测到复位

### WFE（等待事件）

与 WFI 类似，但退出条件增加了：
- `EVENTI` 输入信号被置位
- 事件寄存器已设置（此时 WFE 不进入待机，直接清除事件寄存器）

## 8. RAS 扩展支持

### 缓存 ECC 和奇偶校验

- L1 数据缓存使用 ECC (SECDED) 保护
- L1 指令缓存可配置为 ECC 或奇偶校验
- L2 TLB 使用奇偶校验保护

### 缓存保护行为

- 单位错误 (correctable)：硬件自动纠正，可选择报告
- 双位错误 (uncorrectable)：标记为数据中毒 (data poisoning)

### 未纠正错误和数据中毒

当检测到不可纠正的错误时，受影响的数据被标记为"中毒的"。如果中毒数据被消费（例如被 load 指令读取），则产生同步数据中止。
