> 原文：[Arm Architecture Reference Manual for A-profile architecture (DDI 0487, Version M.b)](https://developer.arm.com/documentation/ddi0487/latest/)
>
> 本文翻译 Part A（架构概述）、Part B（AArch64 应用级架构）、Part D（AArch64 系统级架构）的核心章节。

## Part A：ARM 架构介绍与概述

### A1.1 关于 ARM 架构

本架构参考手册中描述的 ARM 架构定义了一个抽象机器的行为，该抽象机器被称为**处理元素 (Processing Element, PE)**。符合 ARM 架构的实现必须符合所描述的处理元素行为。本手册不打算描述如何构建 PE 的实现，也不限制此类实现的范围（超出已定义的行为之外）。

除非架构另有规定，符合 ARM 架构的实现的程序员可见行为必须与处理元素上程序的简单顺序执行相同。程序员可见行为不包括任何指令的指定执行时间，但要求每条指令在有限时间内执行完成。

ARM 架构包含以下定义：

- 关联的调试架构（AArch64 自托管调试、AArch32 自托管调试、外部调试）
- 关联的跟踪架构（ETMv4、自托管跟踪、嵌入式跟踪扩展、跟踪缓冲区扩展）

ARM 架构是一种**精简指令集计算机 (RISC)** 架构，具有以下 RISC 特征：

- **大的统一寄存器文件**
- **加载/存储架构**：数据处理操作只操作寄存器内容，不直接操作内存内容
- **简单寻址模式**：所有加载/存储地址仅由寄存器内容和指令字段确定

架构定义了 PE 与内存（包括缓存）的交互，并包含内存转换系统。它还描述了多个 PE 之间以及与系统中其他 observer 的交互方式。

本手册定义了 A-profile 架构的版本 8 和版本 9。

ARM 架构支持：

- **64 位执行状态 AArch64**
- **32 位执行状态 AArch32**（与之前版本的 ARM 架构兼容）

两种执行状态都支持浮点指令：

**AArch32 状态提供：**
- 基础指令集中操作 32 位通用寄存器的 SIMD 指令
- 操作 SIMD&FP 寄存器文件的 Advanced SIMD 指令
- 操作 SIMD&FP 寄存器文件的标量浮点指令

**AArch64 状态提供：**
- 操作 SIMD&FP 寄存器文件的 Advanced SIMD 指令
- 操作 SIMD&FP 寄存器文件的标量浮点指令
- 操作 SVE 寄存器文件的 SVE 指令
- 操作 SVE 寄存器文件和 ZA 存储的 SME 指令

> **注意：** A64 指令集不包含操作通用寄存器的 SIMD 指令，因此一些 AArch64 指令描述使用 SIMD 作为 Advanced SIMD 的同义词。

### A1.2 架构配置文件

ARM 架构自引入以来已经显著演进。迄今为止已定义了九个主要版本（1 到 9），其中前三个版本现已过时。

通用名称 AArch64 和 AArch32 描述 64 位和 32 位执行状态：

- **AArch64**：64 位执行状态，地址保存在 64 位寄存器中，基础指令集中的指令可以使用 64 位寄存器处理。支持 A64 指令集。
- **AArch32**：32 位执行状态，地址保存在 32 位寄存器中，基础指令集中的指令使用 32 位寄存器处理。支持 T32 和 A32 指令集。

ARM 定义三种架构配置文件：

```
┌──────────────────────────────────────────────────────────────────┐
│  A-profile (应用配置文件)                                         │
│  - 支持基于 MMU 的虚拟内存系统架构 (VMSA)                          │
│  - 支持 A64, A32, T32 指令集                                     │
│  - 面向：智能手机、服务器、PC 等                                   │
├──────────────────────────────────────────────────────────────────┤
│  R-profile (实时配置文件)                                         │
│  - 支持基于 MPU 的受保护内存系统架构 (PMSA)                        │
│  - 可选支持基于 MMU 的 VMSA                                       │
│  - 支持 A64 或 A32/T32 指令集                                    │
│  - 面向：汽车 ECU、工业控制等                                     │
├──────────────────────────────────────────────────────────────────┤
│  M-profile (微控制器配置文件)                                     │
│  - 为低延迟中断处理设计的编程模型                                  │
│  - 硬件寄存器入栈，支持用高级语言编写中断处理程序                    │
│  - 支持基于 MPU 的 PMSA                                          │
│  - 支持 T32 指令集的变体                                         │
│  - 面向：IoT、传感器、低功耗嵌入式                                 │
└──────────────────────────────────────────────────────────────────┘
```

### A1.3 ARM 架构概念

#### 执行状态 (Execution State)

PE 在任何时刻都处于两种执行状态之一：AArch64 或 AArch32。执行状态决定：

- 支持的寄存器宽度
- 支持的指令集
- 对异常模型的重大方面的影响
- 虚拟内存系统的工作方式

#### 指令集

- **A64**：AArch64 状态使用的指令集，使用 32 位固定长度指令
- **A32**：AArch32 状态中的 ARM 指令集，32 位固定长度
- **T32**：AArch32 状态中的 Thumb 指令集，混合 16 位和 32 位指令

#### 系统寄存器

系统寄存器提供对架构特性的控制和状态信息。在 AArch64 中，通过 `MSR`（写）和 `MRS`（读）指令访问系统寄存器。

---

### A1.4 支持的数据类型

ARM 架构支持以下整数数据类型：

| 类型 | 大小 |
|------|------|
| Byte (字节) | 8 位 |
| Halfword (半字) | 16 位 |
| Word (字) | 32 位 |
| Doubleword (双字) | 64 位 |
| Quadword (四字) | 128 位 |

---

### A1.6 ARM 内存模型

ARM 架构中的内存模型是**弱一致性 (weakly ordered)** 内存模型。这意味着内存访问的顺序不一定与程序顺序匹配。处理器可以对内存访问进行重排序以提高性能。

软件可以使用**内存屏障 (memory barriers)** 来强制特定的顺序保证。

## Part B：AArch64 应用级架构

### B1.1 关于应用级编程模型

应用级编程模型描述了在最低特权级别（EL0）执行的软件可用的特性。通过 AArch64 应用级编程模型，软件可以访问：

- 通用寄存器（X0-X30、SP）
- SIMD&FP 寄存器（V0-V31）
- SVE 可缩放向量寄存器（Z0-Z31）
- SVE 谓词寄存器（P0-P15）
- 特殊寄存器（NZCV、FPCR、FPSR 等）

### B1.2 AArch64 执行状态中的寄存器

```
通用寄存器:
┌─────────────────────────────────────────────────────────────────┐
│ X0-X30  : 64位通用寄存器 (W0-W30 是其低32位视图)                │
│ SP      : 栈指针                                                │
│ PC      : 程序计数器 (不可直接作为通用寄存器访问)                  │
│ XZR/WZR : 零寄存器                                              │
└─────────────────────────────────────────────────────────────────┘

SIMD & 浮点寄存器:
┌─────────────────────────────────────────────────────────────────┐
│ V0-V31  : 128位 SIMD&FP 寄存器                                  │
│           Bn(8位), Hn(16位), Sn(32位), Dn(64位), Qn(128位) 视图 │
└─────────────────────────────────────────────────────────────────┘

SVE 寄存器 (可缩放，IMPLEMENTATION DEFINED 宽度 128-2048位):
┌─────────────────────────────────────────────────────────────────┐
│ Z0-Z31  : SVE 可缩放向量寄存器 (低128位与 V0-V31 别名)          │
│ P0-P15  : SVE 谓词寄存器                                        │
│ FFR     : 首次故障寄存器                                        │
└─────────────────────────────────────────────────────────────────┘
```

### B1.3 进程状态 PSTATE

PSTATE 是进程状态信息的抽象。它不是单个寄存器，而是一组可通过特殊寄存器访问的字段集合：

- **N, Z, C, V**：条件标志（负、零、进位、溢出）
- **SS**：软件单步
- **IL**：非法执行状态
- **nRW**：当前执行状态（0=AArch64，1=AArch32）
- **EL**：当前异常级别
- **SP**：栈指针选择（SP_EL0 或 SP_ELx）
- **D, A, I, F**：调试、SError、IRQ、FIQ 屏蔽位

---

### B2.1 关于 ARM 内存模型

ARM 内存模型定义了内存的行为方式，包括排序规则和一致性保证。

#### 地址空间

在 AArch64 中，虚拟地址空间最大为 52 位（从 Armv8.2-A 起），物理地址空间最大也为 52 位。

#### 内存类型概述

ARM 架构定义两种互斥的内存类型：

- **Normal 内存**：用于代码和数据。可缓存，访问可重排序、合并、推测
- **Device 内存**：用于外设。不可缓存，访问有严格排序限制

### B2.2 ARM 架构中的原子性

#### 单拷贝原子性

如果一次读取获得了一次写入产生的完整值（而非部分值），则该访问具有单拷贝原子性。

#### 多拷贝原子性

Armv8 架构是**多拷贝原子**的 (Other-multi-copy atomic)：一个写入一旦对一个非写入者的 observer 可见，就对所有 observer 可见。

### B2.6 内存屏障

ARM 架构提供三种内存屏障指令：

```
┌────────────────────────────────────────────────────────────────┐
│  DMB (Data Memory Barrier)                                      │
│  - 确保屏障前的内存访问对指定共享域中的其他 observer 可见，       │
│    然后才执行屏障后的内存访问                                    │
├────────────────────────────────────────────────────────────────┤
│  DSB (Data Synchronization Barrier)                             │
│  - 比 DMB 更强：确保屏障前的所有内存访问完成后，                 │
│    屏障后的任何指令才执行                                        │
├────────────────────────────────────────────────────────────────┤
│  ISB (Instruction Synchronization Barrier)                      │
│  - 刷新处理器流水线，确保 ISB 后的所有指令从缓存/内存重新获取     │
│  - 用于确保系统寄存器修改对后续指令可见                           │
└────────────────────────────────────────────────────────────────┘
```

## Part D：AArch64 系统级架构

### D1.1 异常级别

ARM 架构定义了四个异常级别 (Exception Level)，EL0 到 EL3：

```
┌─────────────────────────────────────────────────────┐
│  EL3 - 安全监视器 (Secure Monitor)                   │
│        最高特权，管理安全/非安全世界切换               │
├─────────────────────────────────────────────────────┤
│  EL2 - 虚拟机监控器 (Hypervisor)                     │
│        虚拟化支持，管理虚拟机                         │
├─────────────────────────────────────────────────────┤
│  EL1 - 操作系统内核 (OS Kernel)                      │
│        特权执行，管理系统资源                         │
├─────────────────────────────────────────────────────┤
│  EL0 - 用户应用 (User Application)                   │
│        非特权执行                                    │
└─────────────────────────────────────────────────────┘

异常只能转到相同或更高的 EL
从异常返回只能到相同或更低的 EL
```

每个异常级别有自己的：
- 栈指针 (SP_ELx)
- 异常链接寄存器 (ELR_ELx)
- 保存的程序状态寄存器 (SPSR_ELx)

### D1.2 指令处理的寄存器

在系统级别，除了通用寄存器外，还有大量系统寄存器用于控制 PE 行为：

- **SCTLR_ELx**：系统控制寄存器（MMU 使能、缓存使能等）
- **TCR_ELx**：转换控制寄存器
- **TTBR0_ELx / TTBR1_ELx**：转换表基址寄存器
- **MAIR_ELx**：内存属性间接寄存器
- **VBAR_ELx**：向量基址寄存器（异常向量表基址）
- **ESR_ELx**：异常综合寄存器（异常原因）
- **FAR_ELx**：故障地址寄存器

### D1.5 进程状态 PSTATE（系统级）

在系统级别，PSTATE 包含额外的控制：

- **CurrentEL**：当前异常级别
- **SPSel**：栈指针选择
- **DAIFSet/DAIFClr**：中断屏蔽控制
- **PAN**：特权访问从不（Privileged Access Never）
- **UAO**：用户访问覆盖

### D1.6 异常

异常是需要特权软件处理的事件。异常类型包括：

- **同步异常**：由指令执行直接导致（SVC、未定义指令、数据中止、指令中止等）
- **IRQ**：普通中断
- **FIQ**：快速中断
- **SError**：系统错误（通常是异步外部中止）

异常向量表结构：

```
VBAR_ELx + offset:
┌────────────────────────────────────────────────────────────────┐
│ 偏移量    │ 来源                │ 异常类型                      │
├───────────┼────────────────────┼──────────────────────────────┤
│ 0x000     │ 当前 EL, SP_EL0   │ Synchronous                  │
│ 0x080     │ 当前 EL, SP_EL0   │ IRQ/vIRQ                     │
│ 0x100     │ 当前 EL, SP_EL0   │ FIQ/vFIQ                     │
│ 0x180     │ 当前 EL, SP_EL0   │ SError/vSError               │
├───────────┼────────────────────┼──────────────────────────────┤
│ 0x200     │ 当前 EL, SP_ELx   │ Synchronous                  │
│ 0x280     │ 当前 EL, SP_ELx   │ IRQ/vIRQ                     │
│ 0x300     │ 当前 EL, SP_ELx   │ FIQ/vFIQ                     │
│ 0x380     │ 当前 EL, SP_ELx   │ SError/vSError               │
├───────────┼────────────────────┼──────────────────────────────┤
│ 0x400     │ 低 EL, AArch64    │ Synchronous                  │
│ 0x480     │ 低 EL, AArch64    │ IRQ/vIRQ                     │
│ 0x500     │ 低 EL, AArch64    │ FIQ/vFIQ                     │
│ 0x580     │ 低 EL, AArch64    │ SError/vSError               │
├───────────┼────────────────────┼──────────────────────────────┤
│ 0x600     │ 低 EL, AArch32    │ Synchronous                  │
│ 0x680     │ 低 EL, AArch32    │ IRQ/vIRQ                     │
│ 0x700     │ 低 EL, AArch32    │ FIQ/vFIQ                     │
│ 0x780     │ 低 EL, AArch32    │ SError/vSError               │
└───────────┴────────────────────┴──────────────────────────────┘
```

### D1.7 进入低功耗状态的机制

架构定义了两种进入低功耗状态的指令：

- **WFI (Wait for Interrupt)**：提示 PE 进入低功耗状态直到中断发生
- **WFE (Wait for Event)**：提示 PE 进入低功耗状态直到事件发生

## Part D：自托管调试

### D2.1 关于自托管调试

AArch64 自托管调试允许在 PE 上运行的调试软件控制同一 PE 的调试特性。这包括：

- **断点 (Breakpoints)**：当执行到特定地址时暂停
- **监视点 (Watchpoints)**：当访问特定数据地址时暂停
- **单步执行**：逐条指令执行

调试异常被路由到适当的异常级别，与普通异常使用相同的异常向量表。

### 调试异常使能

调试异常的生成由 `MDSCR_EL1.MDE`（监视器调试使能）和 `MDSCR_EL1.SS`（软件单步）等字段控制。
