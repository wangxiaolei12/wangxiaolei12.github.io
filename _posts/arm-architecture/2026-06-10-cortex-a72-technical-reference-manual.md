# Cortex-A72 技术参考手册

> 原文：[ARM Cortex-A72 MPCore Processor Technical Reference Manual r0p3](https://developer.arm.com/documentation/100095/latest/)
>
> 本文翻译该 TRM 的功能描述核心章节。

---

## 1. 关于 Cortex-A72 处理器

Cortex-A72 处理器是一款**高性能、低功耗**处理器，实现了 ARMv8-A 架构。它在单个处理器器件中包含 1 到 4 个核心，带有 L1 和 L2 缓存子系统。

实现了 ARM GIC v3 架构。

---

## 2. 特性

- 完整实现 ARMv8-A 架构配置文件
- **超标量 (superscalar)、变长、乱序 (out-of-order) 流水线**
- 动态分支预测，带 BTB 和 GHB RAM、返回栈和间接预测器
- 48 条目全关联 L1 指令 TLB，原生支持 4KB、64KB、1MB 页大小
- 32 条目全关联 L1 数据 TLB，原生支持 4KB、64KB、1MB 页大小
- 4 路组相联 1024 条目统一 L2 TLB
- 固定 **48KB L1 指令缓存**和 **32KB L1 数据缓存**
- 共享 L2 缓存，可配置大小：**512KB、1MB、2MB 或 4MB**
- 可选 ECC 保护（L2 缓存、L1 数据缓存、L1 指令缓存奇偶校验）
- AMBA 4 ACE 或 CHI 主接口
- 可选加速器一致性端口 (ACP)，AXI4 从接口
- ETMv4 架构嵌入式跟踪宏单元
- PMUv3 架构性能监视单元
- 跨触发接口 (CTI) 用于多处理器调试
- 可选加密引擎
- 可选 GIC CPU 接口
- 支持多电源域电源管理

### 指令获取

指令获取单元从 L1 指令缓存获取指令，每周期最多向指令解码单元发送 **3 条指令**。支持动态和静态分支预测。

指令获取单元包括：
- **L1 指令缓存**：48KB，3 路组相联，64 字节缓存行，可选双位奇偶校验保护
- **L1 指令 TLB**：48 条目全关联，原生支持 4KB/64KB/1MB 页大小
- **2 级动态预测器**，带 BTB 用于快速目标生成
- 静态分支预测器
- 间接预测器
- 返回栈

### 指令解码

指令解码单元解码 A32、T32 和 A64 指令集，包括 Advanced SIMD 和浮点指令集。

指令解码单元还执行**寄存器重命名**，通过消除 Write-After-Write (WAW) 和 Write-After-Read (WAR) 冒险来促进乱序执行。

### 指令分派

指令分派单元控制解码后的指令何时被分派到执行流水线以及返回结果何时退出。它包括：
- ARM 核心通用寄存器
- Advanced SIMD 和浮点寄存器组
- AArch32 CP15 和 AArch64 系统寄存器

### 整数执行

整数执行单元包括：
- **2 条 ALU 流水线**
- 整数乘累加和 ALU 流水线
- 迭代整数除法硬件
- 分支和指令条件码解析逻辑
- 结果前递和比较器逻辑

### Load/Store 单元

Load/Store 执行单元执行加载和存储指令，包含 L1 数据侧内存系统。它还服务来自 L2 内存系统的内存一致性请求。

包括：
- **L1 数据缓存**：32KB，2 路组相联，64 字节缓存行，可选 ECC 保护（每 32 位）
- **L1 数据 TLB**：32 条目全关联，原生支持 4KB/64KB/1MB 页大小
- **自动硬件预取器**：生成针对 L1D 缓存和 L2 缓存的预取

### L2 内存系统

L2 内存系统服务每个处理器的 L1 指令和数据缓存未命中。管理 ACE/CHI 主接口和可选 ACP 从接口上的请求。

包括：
- **L2 缓存**：512KB/1MB/2MB/4MB 可配置，16 路组相联，可选数据 ECC 保护（每 64 位）
- 每个处理器的 L1 数据缓存 Tag RAM 副本（处理 snoop 请求）
- 每个处理器 **1024 条目 L2 TLB**，4 路组相联
- 自动硬件预取器，可编程指令获取距离

### Advanced SIMD 和浮点单元

提供 ARMv8 Advanced SIMD 和浮点执行支持。此外还提供可选加密引擎支持。

## 4. MMU

### TLB 组织

```
┌────────────────────────────────────────────────────────────────┐
│                          MMU                                     │
│                                                                  │
│  ┌─────────────────────┐    ┌─────────────────────┐            │
│  │   L1 指令 TLB       │    │   L1 数据 TLB       │            │
│  │   48 条目, 全关联   │    │   32 条目, 全关联   │            │
│  │   4KB/64KB/1MB      │    │   4KB/64KB/1MB      │            │
│  └──────────┬──────────┘    └──────────┬──────────┘            │
│             │ 未命中                    │ 未命中                 │
│             └────────────┬──────────────┘                       │
│                          ▼                                       │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │              统一 L2 TLB                                   │ │
│  │   1024 条目, 4路组相联                                    │ │
│  │   支持所有页大小                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                          │ 未命中                                │
│                          ▼                                       │
│                    硬件页表遍历                                   │
└────────────────────────────────────────────────────────────────┘
```

**L1 指令 TLB：**
- 48 条目全关联
- 原生支持 4KB、64KB 和 1MB 页大小
- 可以缓存 Stage 1 和 Stage 2 的组合转换

**L1 数据 TLB：**
- 32 条目全关联
- 原生支持 4KB、64KB 和 1MB 页大小

**L2 TLB：**
- 1024 条目，4 路组相联
- 统一处理指令和数据 TLB 未命中
- 位于每个处理器内部

### TLB 匹配过程

TLB 查找在 L1 TLB 中进行匹配。匹配基于：
- 虚拟地址
- ASID（地址空间标识符）
- VMID（虚拟机标识符）
- 安全状态

如果 L1 TLB 未命中，则查询 L2 TLB。如果 L2 TLB 也未命中，则启动硬件页表遍历。

## 5. 时钟和复位

### 时钟

Cortex-A72 处理器有以下时钟域：
- **核心时钟 (CLK)**：驱动核心逻辑
- **ATCLK**：驱动调试 APB 接口
- **PCLKDBG**：调试 APB 时钟
- **TSCLK**：时间戳时钟

### 复位

处理器支持两种复位：
- **nCPUPORESET**：冷复位，将核心中所有状态复位到其初始值
- **nCORERESET/nRESET**：温复位，复位核心但保留调试寄存器状态

## 6. 电源管理

### 动态电源管理

Cortex-A72 处理器使用多种技术来管理动态功耗：
- 大量使用时钟门控来减少冗余翻转
- 如果指令不需要功能单元，则禁用该功能单元的时钟

### 电源域

处理器定义以下电源域：

```
┌──────────────────────────────────────────────────────────┐
│  PDCORE<n>  — 每个核心的独立电源域                         │
│    包含：核心逻辑、L1 缓存、L1 TLB                        │
│    可独立上下电                                           │
├──────────────────────────────────────────────────────────┤
│  PDL2      — L2 内存系统电源域                            │
│    包含：L2 缓存、snoop 控制、L2 TLB                     │
│    所有核心共享                                           │
├──────────────────────────────────────────────────────────┤
│  PDDEBUG   — 调试电源域                                  │
│    包含：调试逻辑                                         │
└──────────────────────────────────────────────────────────┘
```

### 电源模式

| 模式 | 描述 |
|------|------|
| Run | 全功能运行，所有时钟使能 |
| Standby | WFI/WFE 后进入，大部分时钟停止，状态保持 |
| Retention | 时钟停止，可选保持逻辑状态 |
| Powerdown | 核心完全断电，所有核心状态丢失 |

### 个别核心关电

每个核心可以独立关电。关电序列：
1. 核心执行 WFI 或由系统电源控制器请求
2. 核心停止时钟
3. 可选：保存调试状态
4. 断电核心电源域

恢复时：
1. 上电
2. 释放复位
3. 核心从复位向量启动

## 7. 编程模型

### 执行状态

Cortex-A72 支持两种执行状态：
- **AArch64**：64 位执行状态，使用 A64 指令集
- **AArch32**：32 位执行状态，使用 A32/T32 指令集

### 异常级别

```
EL3 (AArch64) — 安全监视器
    │
    ├─ EL2 (AArch64) — Hypervisor
    │      │
    │      └─ EL1 (AArch64/AArch32) — OS 内核
    │             │
    │             └─ EL0 (AArch64/AArch32) — 用户应用
    │
    └─ Secure EL1 — Trusted OS
           │
           └─ Secure EL0 — Trusted 应用
```

### 安全状态

处理器支持两种安全状态：
- **Secure（安全）**：可以访问安全和非安全资源
- **Non-secure（非安全）**：只能访问非安全资源

EL3 始终在安全状态运行，控制 Secure 和 Non-secure 世界之间的切换。

### 内存模型

Cortex-A72 实现 ARMv8-A 虚拟内存系统架构 (VMSA)：
- 支持 4KB、16KB 和 64KB 转换粒度
- 最多 48 位虚拟地址
- 最多 44 位物理地址（IMPLEMENTATION DEFINED）
- 支持 Stage 1 和 Stage 2 转换
- 支持 ASID 和 VMID

## 8. GIC CPU 接口

处理器包含 GIC v3 CPU 接口（仅 CPU 接口部分）。系统需要外部 GIC 分发器组件。

GIC CPU 接口功能：
- 接收来自外部分发器的中断
- 提供中断确认和优先级处理
- 支持中断虚拟化
- 通过系统寄存器接口访问

## Cortex-A72 vs Cortex-A55 对比

| 特性 | Cortex-A72 | Cortex-A55 |
|------|-----------|-----------|
| 流水线 | 乱序超标量 | 有序 |
| 发射宽度 | 3 条/周期 | 2 条/周期 |
| L1 I-Cache | 48KB, 3路 | 可配置 |
| L1 D-Cache | 32KB, 2路 | 可配置 |
| L1 I-TLB | 48 条目, 全关联 | 10 条目 |
| L1 D-TLB | 32 条目, 全关联 | 10 条目 |
| L2 TLB | 1024 条目, 4路 | 1024 条目, 4路 |
| L2 Cache | 共享, 最大 4MB | 私有, 最大 256KB |
| 定位 | 高性能 (Big) | 高效率 (Little) |
| 分支预测 | 复杂多级 | 简单预测 |
| 寄存器重命名 | 是 | 否 |
